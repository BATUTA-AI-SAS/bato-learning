## Hilo conductor

En 00-10 aprendiste a tomar decisiones sobre un valor. Pero en auditoría rara vez trabajas con un solo valor: trabajas con cientos de facturas, decenas de proveedores, múltiples campos por factura. Python tiene dos estructuras básicas para esto: **listas** y **diccionarios**. Después de este módulo, 00-12 te enseña a recorrerlos con `for` para procesar cada elemento, que es el núcleo de cualquier script de datos.

## Puente desde Excel

| Concepto Python | Equivalente Excel | Descripción |
|-----------------|-------------------|-------------|
| `list` | Una columna de celdas | Secuencia de valores en orden |
| `list[0]` | Primera celda de la columna | El índice en Python empieza en 0, no en 1 |
| `list[1:3]` | Rango de celdas A2:A3 | Slice: devuelve una sublista |
| `dict` | Una fila con encabezados | Clave → valor, como una tabla de 1 fila |
| `dict["campo"]` | `=VLOOKUP("campo", tabla, 2, FALSE)` | Acceso directo por nombre, sin buscar |
| `len(lista)` | `=COUNTA(A1:A100)` | Cuántos elementos tiene |

La diferencia clave con `=VLOOKUP`: el dict no busca nada, accede directamente. Si tienes la clave, el tiempo de acceso es siempre el mismo sin importar cuántas claves tenga el dict.

## Idea central

Una **lista** (`list`) guarda valores en orden, accesibles por su posición numérica. Un **diccionario** (`dict`) guarda pares clave-valor, accesibles por el nombre de la clave. La lista es una columna; el dict es una fila con encabezados. En un dataset de facturas, la lista contiene todas las facturas y cada factura es un dict con sus campos.

## Por qué importa

Sin listas, tendrías que declarar `amount_1 = 4820`, `amount_2 = 1500`, `amount_3 = 3100`... para cada factura. Inviable para cien facturas, imposible para diez mil. Con una lista, iteras sobre todas en tres líneas.

Sin diccionarios, acceder al monto de una factura requiere recordar que está en la columna 4 o en el índice `[3]`. Con un dict, `invoice["amount"]` es legible por cualquier persona que lea el código seis meses después.

## Cómo funciona por dentro

### Listas

Una lista se crea con corchetes `[...]` y comas entre elementos:

```python
amounts = [4820.0, 1500.0, 3100.0, 0.0]
vendors = ["acme_global", "andina_co", "globex_log"]
```

Los elementos pueden ser de cualquier tipo — incluso mezclados, aunque en práctica es mejor mantenerlos del mismo tipo.

**Acceso por índice**: Python cuenta desde `0`, no desde `1`. El primer elemento es `[0]`, el segundo `[1]`, el último es `[-1]`.

```python
amounts[0]   # 4820.0 — primera factura
amounts[1]   # 1500.0 — segunda factura
amounts[-1]  # 0.0    — última factura (cuenta desde el final)
```

> [!cuidado]
> En Excel la primera fila es la fila 1. En Python el primer elemento es el índice 0. Este cambio produce errores de «off-by-one» cuando traduces lógica de Excel a Python. Cuando iteres en 00-12 con `for`, el índice no importa — pero cuando accedes directamente, recuerda: empieza en 0.

**Slice**: devuelve una sublista entre dos posiciones. El segundo índice es **exclusivo** (no se incluye):

```python
amounts[1:3]  # [1500.0, 3100.0] — elementos en posición 1 y 2 (no 3)
amounts[:2]   # [4820.0, 1500.0] — los dos primeros
amounts[2:]   # [3100.0, 0.0]    — desde posición 2 hasta el final
```

**Añadir elementos**: `.append(valor)` añade al final de la lista.

```python
amounts.append(2200.0)
# amounts ahora es [4820.0, 1500.0, 3100.0, 0.0, 2200.0]
```

**Longitud**: `len(lista)` devuelve cuántos elementos tiene.

```python
len(amounts)  # 5
```

**Comprobar si un valor está en la lista**: `in` y `not in`.

```python
if 1500.0 in amounts:
    print("existe esa factura")

if "nuevo_prov" not in vendors:
    print("proveedor no registrado")
```

### Mutabilidad de listas

Una lista es **mutable**: puedes cambiar sus elementos después de crearla.

```python
vendors[0] = "acme_manufacturing"   # reemplaza el primer elemento
vendors.append("initech_svc")       # añade uno nuevo
```

> [!nota]
> La mutabilidad tiene una trampa: si haces `copy = original` y luego cambias `copy`, también cambias `original` — las dos variables apuntan a la misma lista. En Track A profundizarás en por qué ocurre esto. Por ahora, trabaja con listas sin copiarlas.

### Diccionarios

Un dict se crea con llaves `{...}`, y cada par se escribe `clave: valor`:

```python
invoice = {
    "id": "INV-001",
    "vendor": "acme_global",
    "amount": 4820.0,
    "paid": False,
}
```

Las claves son casi siempre strings. Los valores pueden ser de cualquier tipo.

**Acceso por clave**: `dict["clave"]`. Si la clave no existe, Python lanza `KeyError`.

```python
invoice["vendor"]   # "acme_global"
invoice["amount"]   # 4820.0
invoice["paid"]     # False
```

**Añadir o actualizar un campo**:

```python
invoice["paid"] = True          # actualizar campo existente
invoice["period"] = "2026-04"   # añadir campo nuevo
```

**Comprobar si una clave existe**:

```python
if "vendor" in invoice:
    print(invoice["vendor"])

if "discount" not in invoice:
    invoice["discount"] = 0.0
```

Usa siempre `in` antes de acceder a un campo que puede no estar — así evitas `KeyError`.

**Acceso seguro con `.get()`**: devuelve `None` (o un valor por defecto) si la clave no existe, en lugar de lanzar error.

```python
discount = invoice.get("discount")          # None si no existe
discount = invoice.get("discount", 0.0)     # 0.0 si no existe
```

**Lista de diccionarios**: el patrón más común en datos — cada dict es una fila, la lista los agrupa como tabla.

```python
invoices = [
    {"id": "INV-001", "vendor": "acme_global",  "amount": 4820.0},
    {"id": "INV-002", "vendor": "andina_co",     "amount": 1500.0},
    {"id": "INV-003", "vendor": "globex_log",    "amount": 3100.0},
]

# acceder al monto de la segunda factura
invoices[1]["amount"]   # 1500.0
```

## Ejemplo conducido

<!-- INLINE_QUIZ: dict-keyerror -->

El script siguiente carga un pequeño dataset de facturas de ACME y realiza tres operaciones: contar, acceder, añadir.

```python
invoices = [
    {"id": "INV-001", "vendor": "acme_global",  "amount": 4820.0, "paid": True},
    {"id": "INV-002", "vendor": "andina_co",     "amount": 1500.0, "paid": False},
    {"id": "INV-003", "vendor": "globex_log",    "amount": 3100.0, "paid": False},
]

# cuántas facturas hay
total_count = len(invoices)
print(f"facturas: {total_count}")   # 3

# acceder a la primera factura
first = invoices[0]
print(f"primera: {first['id']} — {first['amount']} EUR")

# marcar la segunda factura como pagada
invoices[1]["paid"] = True
print(f"INV-002 paid: {invoices[1]['paid']}")   # True

# añadir una nueva factura
invoices.append({"id": "INV-004", "vendor": "acme_global", "amount": 2200.0, "paid": False})
print(f"facturas tras añadir: {len(invoices)}")   # 4

# acceso seguro a un campo que puede no estar
period = invoices[0].get("period", "sin período")
print(period)   # "sin período"
```

## Determinístico vs agéntico

| Tramo | Tipo | Por qué |
|-------|------|---------|
| Acceder a `invoice["amount"]` | determinístico | Acceso directo por clave; sin ambigüedad |
| Contar facturas con `len()` | determinístico | Operación pura y cerrada |
| Detectar si un campo clave está ausente (`not in`) | determinístico | Regla booleana exacta |
| Decidir si una factura con `amount=0` es un error o una nota de crédito válida | agéntico | El ERP puede emitir ambas con el mismo campo; requiere contexto |

## Errores típicos

**1. Índice fuera de rango.**
*Síntoma*: `IndexError: list index out of range`.
*Causa*: `invoices[5]` cuando la lista solo tiene 3 elementos.
*Arreglo*: comprueba `len(invoices)` antes de acceder por índice, o usa `for` para iterar (00-12).

**2. Clave inexistente en dict.**
*Síntoma*: `KeyError: 'discount'` al ejecutar `invoice["discount"]`.
*Causa*: el campo no existe en ese dict.
*Arreglo*: usa `invoice.get("discount", 0.0)` o comprueba con `if "discount" in invoice:` antes.

**3. Off-by-one: confundir índice 0 con fila 1.**
*Síntoma*: el código devuelve la segunda factura cuando esperabas la primera.
*Causa*: `invoices[1]` en Python es el segundo elemento, no el primero.
*Arreglo*: recuerda que Python empieza en 0. La primera factura es `invoices[0]`.

**4. Modificar la lista mientras se itera sobre ella.**
*Síntoma*: el bucle se salta elementos o lanza `RuntimeError`.
*Causa*: llamar `.append()` o `.remove()` dentro de un `for` que recorre la misma lista.
*Arreglo*: en 00-12 aprenderás el patrón correcto (construir una lista nueva). Por ahora, no modifiques la lista dentro de un `for`.
