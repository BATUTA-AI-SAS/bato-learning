## Puente desde Excel

En Excel hay dos formas de reutilizar trabajo que no escribiste tú:

1. **Función built-in**: `=SUMPRODUCT(...)` siempre estuvo ahí, no la instalaste.
2. **Referencia a otro libro**: `='[OtroLibro.xlsx]Hoja1'!A1` lee una celda de otro archivo Excel.

En Python ocurre lo mismo con dos categorías:

| En Excel | En Python | Cuándo |
|---|---|---|
| Función built-in (`=SUM`, `=LEN`) | Función built-in (`len()`, `print()`) | Siempre disponible, sin `import` |
| Referencia a otro libro | `import csv` (librería estándar) | Viene con Python, sin instalar |
| Complemento descargado del marketplace | `uv add pandas` + `import pandas` | Instalas primero, luego importas |

## Idea central

Un **módulo** es cualquier archivo `.py` que contiene funciones, variables, o clases. `import` carga ese archivo y lo deja disponible. La **librería estándar** es el conjunto de módulos que Python incluye por defecto: `csv`, `json`, `math`, `datetime`, `random`, `os`, `pathlib`. No necesitas instalar nada para usarlos. Las **librerías externas** vienen de PyPI (el repositorio público de paquetes Python) y se instalan con `uv add nombre`.

## Por qué importa

Sin `import` no puedes reutilizar trabajo. Cada script tendría que reimplementar todo desde cero: parseo de fechas, generación de números aleatorios, operaciones matemáticas, lectura de JSON. La librería estándar ya tiene todo eso. Los módulos del Track A en adelante usan docenas de librerías externas (`fastapi`, `sqlalchemy`, `anthropic`). Saber cómo funciona `import` es la base para entender esos módulos.

## Cómo funciona por dentro

### La forma más común: `import módulo`

```python
import math

resultado = math.sqrt(16)   # → 4.0
potencia = math.pow(2, 10)  # → 1024.0
```

`math` se carga como un objeto. Accedes a sus funciones con `math.nombre_función`. El prefijo `math.` dice de dónde viene la función, lo que evita confusiones cuando dos módulos tienen funciones con el mismo nombre.

### Importar un nombre específico: `from módulo import nombre`

```python
from math import sqrt

resultado = sqrt(16)  # ya no necesitas el prefijo math.
```

Útil cuando usas una función muchas veces y el prefijo repetiría. Desventaja: si dos módulos definen `sqrt`, el segundo `from X import sqrt` sobreescribe al primero sin aviso.

### Ejemplo con `random`

```python
import random

vendors = ["ACME", "Globex", "Initech"]
elegido = random.choice(vendors)   # elige uno al azar
print(elegido)

# generar un monto aleatorio entre 100 y 5000
amount = random.uniform(100.0, 5000.0)
print(f"monto: {amount:.2f} EUR")
```

`random` viene con Python: no instalas nada. `random.choice` elige un elemento al azar de una lista; `random.uniform` genera un float entre dos valores.

### Librería estándar vs librería externa

**Librería estándar** — ya instalada con Python:

```python
import csv        # leer y escribir CSV
import json       # leer y escribir JSON
import math       # funciones matemáticas
import datetime   # fechas y horas
import os         # interacción con el sistema operativo
import pathlib    # rutas de archivos de forma moderna
import random     # generación de números aleatorios
```

**Librería externa** — necesitas instalarla primero:

```python
# primero en terminal: uv add pandas
import pandas     # DataFrames, análisis de datos

# primero: uv add fastapi
import fastapi    # servidor web

# primero: uv add anthropic
import anthropic  # SDK de Anthropic para LLMs
```

> [!nota]
> En internet encontrarás muchos ejemplos con `pip install pandas`. En este curso usamos `uv add pandas`. El resultado es el mismo —la librería queda instalada— pero `uv` gestiona también las versiones exactas de cada paquete para que el proyecto sea reproducible en cualquier máquina. Cuando veas `pip install` en un tutorial, mentalmente tradúcelo a `uv add`.

### Qué pasa cuando escribes `import math`

Python busca `math` en una lista ordenada de lugares llamada `sys.path`. Primero busca en la librería estándar; luego en las librerías instaladas en el entorno virtual del proyecto; luego en el directorio actual. Si no lo encuentra en ninguno, lanza `ModuleNotFoundError`.

No necesitas conocer `sys.path` de memoria ahora. Lo importante es saber que `import csv` funciona sin instalación (librería estándar) y `import pandas` requiere `uv add pandas` primero (externa).

> [!nota]
> El sistema completo de módulos, paquetes, `__init__.py`, imports relativos, y `sys.path` se enseña en el módulo A05 del Track A. Aquí solo necesitas saber usar `import` para acceder a funciones ya escritas.

## Ejemplo conducido

<!-- INLINE_QUIZ: stdlib-vs-external -->

Un script de simulación que genera facturas de prueba para ACME usando solo librería estándar:

```python
import random
import csv
import io

vendors = ["ACME", "Globex", "Initech"]
statuses = ["PAID", "OVERDUE", "PENDING"]

# generar 5 facturas de prueba
invoices = []
for i in range(1, 6):
    invoices.append({
        "id": f"INV-{i:03d}",
        "vendor": random.choice(vendors),
        "amount": round(random.uniform(500.0, 5000.0), 2),
        "status": random.choice(statuses),
    })

# mostrar como tabla CSV
output = io.StringIO()
writer = csv.DictWriter(output, fieldnames=["id", "vendor", "amount", "status"])
writer.writeheader()
writer.writerows(invoices)
print(output.getvalue())
```

`random`, `csv`, e `io` son librería estándar: cero instalaciones. El script genera datos de prueba reproducibles en cualquier máquina con Python 3.12.

## Errores típicos

**1. `ModuleNotFoundError: No module named 'pandas'`.**
*Síntoma*: `import pandas` falla aunque acabas de ver que alguien lo usa.
*Causa*: `pandas` es librería externa y no está instalada en este entorno.
*Arreglo*: `uv add pandas` en la terminal del proyecto, luego vuelve a ejecutar el script.

**2. `ModuleNotFoundError: No module named 'csv'` (nunca debería pasar).**
*Síntoma*: el error apunta a una librería estándar.
*Causa*: nombre mal escrito (`improt csv`, `import CSV`, `import Csv`). Python distingue mayúsculas.
*Arreglo*: corriges el typo. Los módulos de librería estándar siempre van en minúscula.

**3. `from math import *` — importar todo sin nombre.**
*Síntoma*: el código funciona, pero si otro módulo también importa `*`, los nombres se sobreescriben en silencio.
*Causa*: `import *` trae todos los nombres del módulo al espacio de nombres actual sin prefijo.
*Arreglo*: importa solo lo que necesitas: `from math import sqrt, pi`. O usa `import math` y el prefijo. El `import *` se considera mala práctica en código de producción.
