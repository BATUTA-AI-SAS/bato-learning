## Hilo conductor

En 00-14 aprendiste a manejar errores cuando los datos vienen mal formados. Ahora vas a cargar esos datos desde un archivo de texto. El archivo más familiar para alguien que trabaja con Excel es el `.csv`: filas de valores separados por coma. Python lo abre, lo lee línea a línea, y lo cierra. En 00-16 importarás módulos de la librería estándar; aquí ya usas uno: `csv`.

## Puente desde Excel

Cuando abres un archivo `.csv` en Excel, Excel lo carga completo en memoria, muestra las columnas, y bloquea el archivo para que nadie más lo escriba al mismo tiempo. Cuando cierras el archivo, Excel lo libera.

Python hace lo mismo pero en código:

```python
with open("invoices.csv", "r") as f:
    contenido = f.read()
# aquí el archivo ya está cerrado
```

El bloque `with` es el equivalente a «abrir el archivo». Cuando termina el bloque (cuando la indentación regresa al nivel anterior), Python cierra el archivo automáticamente — sin que tengas que recordarlo, igual que Excel lo cierra al salir.

## Idea central

`open(ruta, modo)` abre un archivo y devuelve un objeto de tipo archivo. El **modo** controla qué puedes hacer: `"r"` para leer, `"w"` para escribir (sobreescribe si existe), `"a"` para añadir líneas al final. El bloque `with` garantiza que el archivo se cierra aunque ocurra un error dentro del bloque. Sin `with`, si tu código falla a mitad, el archivo puede quedar abierto y bloqueado.

## Por qué importa

Un script que lee archivos CSV es la primera herramienta real de automatización para alguien que trabaja con datos de negocio. El equipo de ACME tiene archivos de facturas exportados de su ERP. Con `open()` y el módulo `csv`, puedes leerlos, filtrarlos, y generar un resumen sin abrir Excel. El resultado puede escribirse en otro archivo para que otro proceso lo consuma. Esto es la base de cualquier pipeline de datos: leer → transformar → escribir.

## Cómo funciona por dentro

### Leer todo el contenido de una vez

```python
with open("clientes.txt", "r") as f:
    content = f.read()

print(content)
```

`f.read()` carga todo el archivo como un único string. Útil para archivos pequeños. Para archivos grandes, es mejor leer línea por línea.

### Leer línea a línea

```python
with open("clientes.txt", "r") as f:
    lines = f.readlines()

# lines es una lista de strings, uno por línea
for line in lines:
    print(line.strip())  # .strip() elimina el salto de línea al final
```

O de forma más directa, iterando el archivo directamente:

```python
with open("clientes.txt", "r") as f:
    for line in f:
        print(line.strip())
```

Iterar el objeto de archivo línea a línea es más eficiente con archivos grandes: Python no carga todo en memoria de golpe.

### Escribir en un archivo

```python
with open("salida.txt", "w") as f:
    f.write("Reporte ACME - Período 2026-04\n")
    f.write("Total facturas: 12\n")
    f.write("Monto total: 48210.50 EUR\n")
```

El modo `"w"` crea el archivo si no existe. Si ya existe, lo sobreescribe desde cero. El modo `"a"` añade líneas al final sin borrar lo que ya hay.

> [!cuidado]
> `"w"` borra el contenido anterior del archivo sin preguntar. Si necesitas preservar lo existente, usa `"a"`. Si quieres verificar que el archivo existe antes de escribir, usa `pathlib.Path` (eso llega en módulos posteriores).

### Leer un CSV como tabla con encabezados

El módulo `csv` de la librería estándar de Python convierte cada fila del CSV en un diccionario, usando la primera fila como nombres de columna. Esto es equivalente a cómo Excel trata los encabezados de columna.

```python
import csv

with open("invoices.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["vendor"], row["amount"])
```

Si el archivo `invoices.csv` tiene este contenido:

```
vendor,amount,status
ACME,4820.50,PAID
Globex,1500.00,OVERDUE
Initech,3100.75,PAID
```

El bucle imprime:

```
ACME 4820.50
Globex 1500.00
Initech 3100.75
```

Cada `row` es un diccionario donde las claves son los nombres de columna de la primera fila.

> [!nota]
> `csv` ya está incluido en Python — no necesitas instalar nada. Es parte de la **librería estándar**: el conjunto de módulos que Python incluye por defecto. En 00-16 verás la distinción entre librería estándar y librerías externas.

### Escribir un CSV

```python
import csv

rows = [
    {"vendor": "ACME", "amount": 4820.50, "status": "PAID"},
    {"vendor": "Globex", "amount": 1500.00, "status": "OVERDUE"},
]

with open("output.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["vendor", "amount", "status"])
    writer.writeheader()
    writer.writerows(rows)
```

`newline=""` en el `open` evita líneas en blanco duplicadas en Windows. Es un detalle técnico que hay que recordar al escribir CSVs.

## Ejemplo conducido

<!-- INLINE_QUIZ: with-open-purpose -->

ACME necesita saber cuántas facturas del período `2026-04` están vencidas (`OVERDUE`). El archivo `facturas.csv` tiene las columnas `vendor`, `amount`, `status`, `period`.

En Pyodide (el ejercicio de abajo simula el archivo en memoria). El mismo código funciona en tu equipo con un archivo real:

```python
import csv
import io

# simulación del archivo (en tu equipo sería open("facturas.csv", "r"))
raw = """vendor,amount,status,period
ACME,4820.50,PAID,2026-04
Globex,1500.00,OVERDUE,2026-04
Initech,3100.75,PAID,2026-04
ACME,800.00,OVERDUE,2026-04
Globex,2200.00,PAID,2026-03
"""

overdue_count = 0
overdue_total = 0.0

reader = csv.DictReader(io.StringIO(raw))
for row in reader:
    if row["status"] == "OVERDUE" and row["period"] == "2026-04":
        overdue_count += 1
        overdue_total += float(row["amount"])

print(f"Facturas vencidas: {overdue_count}")
print(f"Monto vencido: {overdue_total:.2f} EUR")
# → Facturas vencidas: 2
# → Monto vencido: 2300.00 EUR
```

> [!nota]
> `io.StringIO(raw)` convierte un string en un objeto que se comporta como un archivo. Es el truco para usar `csv.DictReader` en los ejercicios del navegador sin necesitar un archivo real en disco. En tu equipo local usas `open("facturas.csv", "r")` directamente.

## Errores típicos

**1. Olvidar `.strip()` en líneas leídas con `readlines()`.**
*Síntoma*: comparas `line == "OVERDUE"` y nunca es `True`, aunque el dato está ahí.
*Causa*: cada línea leída con `readlines()` incluye `\n` al final. El string `"OVERDUE\n"` no es igual a `"OVERDUE"`.
*Arreglo*: `line.strip()` elimina espacios y saltos de línea del inicio y el final.

**2. Abrir el archivo en modo `"w"` cuando querías `"a"`.**
*Síntoma*: el archivo existe con datos y después de ejecutar el script está vacío o solo tiene las últimas líneas escritas.
*Causa*: `"w"` sobreescribe desde cero.
*Arreglo*: `"a"` para añadir, `"w"` solo cuando quieres reemplazar el archivo completo.

**3. No usar `with` y olvidar cerrar el archivo.**
*Síntoma*: el archivo queda abierto; en Windows puede quedar bloqueado para otros procesos o para el mismo script la siguiente vez.
*Causa*: `f = open(...)` sin `with` no cierra automáticamente.
*Arreglo*: usa siempre `with open(...) as f:`. Es la forma idiomática y segura.
