## Hilo conductor

En 00-12 escribiste bucles que calculaban totales y filtraban facturas. Si ese mismo cálculo se necesita en tres partes distintas del script, lo copias tres veces — y cuando hay que cambiar un umbral, lo cambias en tres sitios. Las funciones resuelven esto: defines la lógica una vez con `def` y la llamas donde la necesites. Después de este módulo, 00-14 enseña `try/except` para manejar errores dentro de funciones de forma controlada.

## Puente desde Excel

Excel 365 introdujo `=LAMBDA()` precisamente porque las fórmulas repetidas en cientos de celdas son imposibles de mantener. Con `LAMBDA`, defines la lógica una vez y la reutilizas con un nombre.

```
Excel 365:
=LET(
  calcular_iva, LAMBDA(monto, monto * 0.21),
  calcular_iva(4820)
)
→ 1012.2

Python — equivalente y más legible:
def compute_iva(amount):
    return amount * 0.21

compute_iva(4820)   # 1012.2
```

| Excel | Python | Nota |
|-------|--------|------|
| `=LAMBDA(x, x*0.21)` | `def f(x): return x * 0.21` | Python nombra la función directamente |
| Definir con nombre en Administrador de nombres | `def nombre(params):` | Mismo concepto — un nombre para reusar |
| Parámetros de `LAMBDA` | Parámetros de `def` | Ambos son los «huecos» que rellenas al llamar |
| El resultado aparece en la celda | `return valor` | Lo que la función devuelve al que la llamó |

La diferencia principal: `LAMBDA` en Excel solo acepta una expresión. Una función Python puede tener tantas líneas como necesites, con condicionales, bucles y llamadas a otras funciones.

## Idea central

Una **función** es un bloque de código con nombre que recibe **parámetros** (los datos que le pasas), ejecuta su lógica y devuelve un **valor** con `return`. Es «una fórmula que tú defines y puedes llamar en cualquier momento». Una función que calcula siempre lo mismo para la misma entrada — sin depender de variables externas ni modificar nada fuera — se llama **función pura**. Es el patrón que usarás casi siempre en este curso.

## Por qué importa

Cuando clasificas el nivel de una factura con `if/elif/else` (00-10), ese bloque funciona para un solo monto. Si mañana tienes cien facturas y quieres clasificar cada una, copias el bloque dentro del `for` — y si los umbrales cambian, los cambias en el `for`. Con una función, lo cambias en un solo lugar y todas las llamadas se actualizan automáticamente.

El pipeline de auditoría de ACME tiene tres operaciones repetidas en distintas partes: calcular IVA, clasificar nivel, y calcular el total de una lista. Las tres son candidatas perfectas para convertirse en funciones.

## Cómo funciona por dentro

### Estructura de una función

```python
def compute_iva(amount):
    return amount * 0.21
```

- `def` indica que estás definiendo una función.
- `compute_iva` es el nombre. Sigue las mismas reglas que las variables: minúsculas, snake_case.
- `amount` es el parámetro: el nombre que usa la función internamente para referirse al valor que recibe.
- Los dos puntos `:` son obligatorios, como en `if` y `for`.
- `return` devuelve el resultado al código que llamó a la función.
- El cuerpo va indentado 4 espacios.

### Llamar a una función

Llamas a la función escribiendo su nombre seguido de paréntesis con los argumentos:

```python
result = compute_iva(4820.0)   # 1012.2
print(result)
```

El argumento `4820.0` se asigna al parámetro `amount` dentro de la función. La función devuelve `4820.0 * 0.21 = 1012.2`. Ese valor queda en `result`.

Puedes llamarla tantas veces como quieras con distintos argumentos:

```python
compute_iva(1500.0)   # 315.0
compute_iva(3100.0)   # 651.0
```

### Múltiples parámetros

```python
def compute_tax(amount, rate):
    return amount * rate
```

Al llamarla, los argumentos se asignan en el mismo orden que los parámetros:

```python
compute_tax(4820.0, 0.21)   # amount=4820.0, rate=0.21 → 1012.2
compute_tax(1500.0, 0.10)   # amount=1500.0, rate=0.10 → 150.0
```

### `return` devuelve el control

El `return` termina la función inmediatamente y envía el valor al llamador. Una función puede tener varios `return` (uno por rama de un `if`):

```python
def get_level(amount):
    if amount > 1000:
        return "alto"
    elif amount > 100:
        return "medio"
    else:
        return "bajo"
```

Cuando Python ejecuta `return "alto"`, la función termina ahí. Las líneas siguientes de la función no se ejecutan para ese caso.

### Función sin `return` devuelve `None`

Si una función no tiene `return`, o llega al final sin encontrarlo, devuelve automáticamente `None`:

```python
def print_invoice(invoice):
    print(f"{invoice['id']}: {invoice['amount']} EUR")
    # no hay return — devuelve None

result = print_invoice({"id": "INV-001", "amount": 4820.0})
print(result)   # None
```

Esto es correcto cuando la función produce un efecto (imprimir, guardar) y no necesita devolver un valor calculado.

<details>
<summary>Para profundizar: función pura vs función con efectos (opcional)</summary>

Una **función pura** solo opera sobre sus parámetros y devuelve un resultado. No imprime, no modifica variables externas, no accede a archivos. Para la misma entrada, siempre produce la misma salida.

```python
# función pura: misma entrada → misma salida, sin efectos
def compute_iva(amount):
    return amount * 0.21

# función con efecto: imprime, no devuelve nada útil
def print_iva(amount):
    print(f"IVA: {amount * 0.21}")
```

Las funciones puras son más fáciles de testear y de reutilizar. Cuando puedas elegir, hazla pura.

</details>

<details>
<summary>Para profundizar: funciones que reciben listas y dicts (opcional)</summary>

Las funciones pueden recibir cualquier tipo: listas, dicts, strings, números.

```python
def sum_invoices(invoices):
    total = 0.0
    for inv in invoices:

batch = [
    {"id": "INV-001", "amount": 4820.0},

sum_invoices(batch)   # 6320.0
```

</details>

<details>
<summary>Para profundizar: *args y **kwargs — funciones que reciben cualquier cantidad de cosas (opcional)</summary>

Una función puede declarar `*args` para recibir cualquier número de argumentos posicionales, y `**kwargs` para recibir cualquier número de argumentos con nombre. Ejemplo:

```python
def log(*args, **kwargs):
    print(args, kwargs)

log(1, 2, 3, nombre="ACME")  # (1, 2, 3) {'nombre': 'ACME'}
```

Útil para wrappers y decoradores. En funciones de dominio normales — como las que escribes en este track — no los necesitas: define los parámetros por nombre.

</details>

<details>
<summary>Para profundizar: convenciones de naming para funciones (opcional)</summary>

Las mismas reglas que para variables: minúsculas, `snake_case`, en inglés. Los nombres de función deben ser verbos o frases verbales que describan qué hace: `compute_iva`, `get_level`, `sum_valid_invoices`. Evita nombres genéricos como `do_thing` o `process`. Las funciones booleanas suelen empezar por `is_` o `has_`: `is_paid`, `has_overdue_invoices`.

</details>

<details>
<summary>Para profundizar: el pipeline completo con tres funciones (opcional)</summary>

Tres funciones para el pipeline de ACME. Cada una hace una sola cosa.

```python
def compute_iva(amount):
    """Calcula el IVA al 21% sobre un monto."""
    return amount * 0.21

def get_level(amount):
    """Clasifica el monto en alto/medio/bajo."""
    if amount > 1000:
        return "alto"
    elif amount > 100:
        return "medio"
    else:
        return "bajo"

def sum_valid_invoices(invoices):
    """Suma los montos de facturas con monto mayor que cero."""
    total = 0.0
    for inv in invoices:
        if inv["amount"] > 0:
            total += inv["amount"]
    return total

invoices = [
    {"id": "INV-001", "vendor": "acme_global",  "amount": 4820.0},
    {"id": "INV-002", "vendor": "andina_co",     "amount": 1500.0},
    {"id": "INV-003", "vendor": "globex_log",    "amount": 0.0},
]

total = sum_valid_invoices(invoices)
print(f"total válido: {total} EUR")   # 6320.0 EUR

for inv in invoices:
    level = get_level(inv["amount"])
    iva = compute_iva(inv["amount"])
    print(f"{inv['id']}: nivel={level}, IVA={iva:.2f} EUR")
```

Fíjate: la lógica está en un solo lugar. Si los umbrales cambian, modificas la función — no el bucle.

</details>

## Determinístico vs agéntico

| Tramo | Tipo | Por qué |
|-------|------|---------|
| `compute_iva(amount)` — tasa fija 21% | determinístico | Función pura; misma entrada → mismo resultado siempre |
| `get_level(amount)` — umbrales numéricos fijos | determinístico | Regla cerrada; los umbrales están en el código |
| `sum_valid_invoices(invoices)` — suma acumulativa | determinístico | Aritmética pura; sin contexto externo |
| Decidir si los umbrales de `get_level` son los correctos para este cliente | agéntico | Los umbrales cambian por industria, período y criterio del cliente; no hay una regla universal |

## Errores típicos

**1. Olvidar `return`.**
*Síntoma*: `result = mi_funcion(...)` y `result` es `None`.
*Arreglo*: añade `return valor`. Si la función imprime, la función calcula y devuelve — el llamador imprime.

**2. Llamar a la función antes de definirla.**
*Síntoma*: `NameError: name 'compute_iva' is not defined`.
*Arreglo*: define la función antes de llamarla en el script.

<details>
<summary>Para profundizar: más errores frecuentes con funciones (opcional)</summary>

**Reutilizar el nombre del parámetro como variable global.**
*Síntoma*: modificar `amount` dentro de la función no cambia nada fuera.
*Causa*: los parámetros son locales a la función — no tocan variables externas del mismo nombre.

**Función que hace demasiado.**
*Síntoma*: la función tiene 30 líneas, calcula, imprime, modifica una lista y llama a otra función.
*Arreglo*: una función, un trabajo. Tres funciones simples son más mantenibles que una compleja.

</details>
