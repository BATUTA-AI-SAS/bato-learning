## Hilo conductor

En 00-09 ejecutaste tu primer script con variables y `print`. Todas las líneas corrían siempre, de arriba abajo. Ahora el script toma decisiones: ejecuta unas líneas y salta otras según el valor de una variable. Eso es un **condicional**. Después de esto, 00-11 introduce listas y diccionarios, y los condicionales que aprendes aquí servirán para filtrar elementos dentro de esas estructuras.

## Puente desde Excel

En Excel escribes `=IF(monto>1000, "alto", IF(monto>100, "medio", "bajo"))`. Funciona, pero anidar `IF` dentro de `IF` produce fórmulas que nadie quiere mantener. En Python, la misma lógica se escribe en cascada, con una línea para cada rama:

```
Excel:
=IF(monto>1000, "alto", IF(monto>100, "medio", "bajo"))

Python:
if monto > 1000:
    category = "alto"
elif monto > 100:
    category = "medio"
else:
    category = "bajo"
```

Cada `elif` es un «si no se cumplió lo anterior, prueba esto». `else` es el «en cualquier otro caso». Python solo entra en la **primera** rama cuya condición sea verdadera; las demás se saltan.

| Excel | Python | Nota |
|-------|--------|------|
| `=IF(A1>0, ...)` | `if amount > 0:` | — |
| `IF(A1>0, ..., IF(A1>-100, ...))` | `if ... elif ... else:` | Más legible |
| `Y(A1>0, A1<1000)` | `amount > 0 and amount < 1000` | |
| `O(A1<0, A1>9000)` | `amount < 0 or amount > 9000` | |
| `NO(A1=0)` | `amount != 0` o `not (amount == 0)` | |

## Idea central

Un **condicional** evalúa una expresión que produce `True` o `False` y ejecuta el bloque de código que corresponde. En Python esa expresión puede combinar cualquier número de comparaciones con `and`, `or` y `not`. El bloque que ejecutar va **indentado 4 espacios** — esa indentación no es decoración, es la sintaxis que Python usa para saber dónde empieza y dónde termina cada rama.

## Por qué importa

Sin condicionales, un script aplica la misma operación a todo. Con condicionales, puedes clasificar facturas, detectar anomalías, aplicar reglas distintas por proveedor o período. Es la diferencia entre una hoja que suma todo y una que distingue categorías.

En el sistema de auditoría de ACME Manufacturing, el primer paso de cualquier pipeline es clasificar las facturas por monto para decidir qué nivel de revisión requieren. Esa clasificación es un bloque `if/elif/else`.

## Cómo funciona por dentro

### Operadores de comparación

| Operador | Significado | Ejemplo |
|----------|-------------|---------|
| `==` | igual a | `vendor == "acme"` |
| `!=` | distinto de | `status != "paid"` |
| `<` | menor que | `amount < 0` |
| `>` | mayor que | `amount > 1000` |
| `<=` | menor o igual | `amount <= 100` |
| `>=` | mayor o igual | `amount >= 100` |

> [!cuidado]
> `=` asigna un valor a una variable: `amount = 4820`. `==` compara dos valores: `amount == 4820`. Mezclarlos es el primer error que comete todo programador nuevo. Python levantará un `SyntaxError` si intentas usar `=` dentro de un `if`, así que lo detectas rápido.

### La indentación es sintaxis

Python usa la indentación para delimitar bloques. Un bloque `if` va seguido por líneas con 4 espacios de sangría. Cuando la indentación vuelve al nivel anterior, el bloque terminó.

```python
amount = 4820.0

if amount > 1000:
    category = "alto"    # 4 espacios — dentro del if
    note = "requiere revisión"  # también dentro del if
# aquí ya estamos fuera del if
print(category)
```

VS Code inserta los 4 espacios automáticamente al pulsar `Enter` después de los dos puntos `:`.

### `elif` y `else`

Python evalúa las condiciones en orden. En cuanto una es `True`, ejecuta ese bloque y **salta todas las demás**. Si ninguna es `True` y existe `else`, ejecuta `else`.

```python
amount = 350.0

if amount > 1000:
    category = "alto"
elif amount > 100:
    category = "medio"   # amount=350 cumple esta condición
else:
    category = "bajo"    # solo si amount <= 100

print(category)  # "medio"
```

### Operadores booleanos

`and`, `or` y `not` combinan condiciones:

```python
# and: las dos tienen que ser verdaderas
if amount > 0 and amount < 500:
    category = "pequeño-positivo"

# or: basta con que una sea verdadera
if amount < 0 or amount > 100000:
    flag = "revisar"

# not: invierte la condición
if not vendor:          # si vendor es cadena vacía "" o None
    flag = "sin proveedor"
```

> [!nota]
> `not vendor` es `True` cuando `vendor` es `""`, `None`, `0`, o cualquier valor «vacío». En Python, una cadena vacía se comporta como `False` en contexto booleano. Aprenderás los detalles en Track A; por ahora úsalo para detectar campos sin rellenar.

### Condicionales anidados

Puedes poner un `if` dentro de otro. Úsalo solo cuando la lógica lo exige — si tienes más de dos niveles, considera reorganizar.

```python
amount = 4820.0
vendor = "acme_global"

if amount > 0:
    if vendor == "acme_global":
        priority = "normal"
    else:
        priority = "revisar_proveedor"
else:
    priority = "factura_nula"
```

## Ejemplo conducido

<!-- INLINE_QUIZ: elif-order -->

El equipo de ACME clasifica las facturas del período `"2026-04"` en tres niveles según el monto, y añade una alerta extra para facturas de proveedores nuevos.

```python
amount = 4820.0
vendor = "andina_co"
known_vendors = ["acme_global", "globex_log"]

# clasificar por monto
if amount > 1000:
    level = "alto"
elif amount > 100:
    level = "medio"
else:
    level = "bajo"

# alerta si el proveedor no es conocido
if vendor not in known_vendors:
    alert = f"proveedor nuevo: {vendor}"
else:
    alert = ""

print(f"nivel={level}  alerta={alert!r}")
# nivel=alto  alerta='proveedor nuevo: andina_co'
```

`not in` comprueba que un valor **no** esté en una lista. Es el equivalente a `=COUNTIF(rango, criterio)=0` en Excel.

## Determinístico vs agéntico

| Tramo | Tipo | Por qué |
|-------|------|---------|
| Clasificar monto en alto/medio/bajo con umbrales fijos | determinístico | Regla cerrada; misma entrada → mismo resultado siempre |
| Detectar si el proveedor está en la lista de conocidos | determinístico | Igualdad de string; no hay ambigüedad |
| Decidir si una factura de proveedor nuevo es fraude o proveedor legítimo reciente | agéntico | Requiere contexto del negocio; la regla no puede ser cerrada |

## Errores típicos

**1. Usar `=` en lugar de `==` en la condición.**
*Síntoma*: `SyntaxError: invalid syntax` en la línea del `if`.
*Causa*: `if amount = 4820:` — se usó el operador de asignación donde va el de comparación.
*Arreglo*: `if amount == 4820:`.

**2. Indentación inconsistente.**
*Síntoma*: `IndentationError: unexpected indent` o código que no se ejecuta cuando debería.
*Causa*: mezclar espacios y tabulaciones, o usar 2 espacios en unas líneas y 4 en otras.
*Arreglo*: configura VS Code para insertar 4 espacios al presionar `Tab` (la configuración por defecto). Nunca mezcles.

**3. Condición que siempre es `True` porque el orden importa.**
*Síntoma*: el `elif` nunca se ejecuta aunque el valor debería cumplirlo.
*Causa*:
```python
if amount >= 0:       # esto captura todo incluyendo amount=350
    category = "bajo"
elif amount > 100:    # nunca llega aquí para amount=350
    category = "medio"
```
*Arreglo*: ordena las condiciones de la más restrictiva a la menos restrictiva, o ajusta los operadores.

**4. Olvidar los dos puntos `:`.**
*Síntoma*: `SyntaxError: expected ':'`.
*Causa*: `if amount > 1000` sin `:` al final.
*Arreglo*: `if amount > 1000:` — los dos puntos son parte de la sintaxis de Python.
