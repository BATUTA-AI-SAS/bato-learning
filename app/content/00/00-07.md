## Hilo conductor

Tienes variables con valores. El siguiente paso es operar con ellas: calcular un total, aplicar un porcentaje, construir el texto de un informe. Python tiene los mismos operadores que Excel para aritmética, más dos que Excel no tiene de forma directa: división entera (`//`) y módulo (`%`). Para texto, la concatenación con `+` existe pero las **f-strings** son la herramienta estándar desde Python 3.6.

## Puente desde Excel

En Excel escribes `=A1+B1` para sumar dos celdas, `=A1*0.21` para calcular IVA, o `=A1^2` para elevar al cuadrado. En Python es casi idéntico: `amount + fee`, `amount * 0.21`, `amount ** 2`. El único cambio de notación es el exponente (`^` en Excel, `**` en Python). Para texto, Excel tiene `=CONCAT(A1, " EUR")` y el operador `&`; Python tiene `+` entre strings y las f-strings, que son más expresivas.

## Idea central

Python tiene siete operadores aritméticos: `+`, `-`, `*`, `/`, `//`, `%`, `**`. Los cuatro primeros funcionan igual que en Excel. `**` es la potencia. `//` es la **división entera** (devuelve el cociente sin decimales). `%` es el **módulo** (devuelve el resto de la división). Para strings, `+` concatena y `*` repite. Las **f-strings** (`f"texto {variable} texto"`) insertan valores dentro de un string de forma directa y legible.

## Por qué importa

Los cálculos financieros de ACME usan estos operadores a diario: monto base × tipo impositivo, número de facturas ÷ número de períodos, resto de la división para saber si un número es par. Saber qué devuelve cada operador y cuándo aparece un `TypeError` evita errores silenciosos en los informes.

Las f-strings son la forma estándar de construir líneas de log, mensajes de error, y reportes textuales en Python moderno. Cualquier código que uses en Tracks A–E las utiliza. El patrón `f"factura {invoice_id} por {amount:.2f} EUR"` aparece decenas de veces en el curso.

## Cómo funciona por dentro

### Operadores aritméticos

| Operador | Descripción | Ejemplo | Resultado | Excel equiv. |
|---|---|---|---|---|
| `+` | suma | `4820 + 180` | `5000` | `=A1+B1` |
| `-` | resta | `5000 - 4820` | `180` | `=A1-B1` |
| `*` | multiplicación | `4820 * 0.21` | `1012.2` | `=A1*0.21` |
| `/` | división real | `5000 / 4` | `1250.0` | `=A1/4` (devuelve float) |
| `//` | división entera | `5000 // 4` | `1250` | parte entera de `=INT(A1/4)` |
| `%` | módulo (resto) | `5000 % 4` | `0` | `=MOD(A1,4)` |
| `**` | potencia | `2 ** 10` | `1024` | `=A1^10` |

Diferencia importante: `/` entre dos `int` devuelve siempre `float`. `5 / 2` es `2.5`, no `2`. Si quieres el entero del cociente, usa `//`: `5 // 2` es `2`.

### Tipos y operaciones: dónde aparece `TypeError`

Python no convierte tipos automáticamente. Si intentas sumar un número y un string, falla:

```python
amount = 4820.0
label = "EUR"
total = amount + label  # TypeError: unsupported operand type(s) for +: 'float' and 'str'
```

Si el número viene como string (por ejemplo, de un CSV o de una celda de Excel exportada), conviértelo primero:

```python
raw = "4820"
amount = float(raw)      # convierte str → float
count = int("7")         # convierte str → int
```

### Concatenación de strings

`+` entre dos strings los une:

```python
prefix = "INV-"
number = "042"
invoice_id = prefix + number   # "INV-042"
```

El operador `*` repite un string:

```python
separator = "-" * 40   # "----------------------------------------"
```

> [!cuidado]
> `"4820" + 100` falla con `TypeError`. Python no convierte el string a número ni el número a string automáticamente. Tienes que hacerlo tú: `int("4820") + 100` o `"4820" + str(100)`.

### F-strings

Una **f-string** es un string que empieza con `f` antes de la comilla. Dentro de llaves `{}` puedes poner cualquier expresión Python; se evalúa y su resultado se inserta en el texto:

```python
invoice_id = "INV-042"
amount = 9350.0
period = "2026-04"

line = f"Factura {invoice_id} del período {period}: {amount} EUR"
# → "Factura INV-042 del período 2026-04: 9350.0 EUR"
```

Para formatear floats con dos decimales, usa el especificador `:.2f` dentro de las llaves:

```python
formatted = f"Total: {amount:.2f} EUR"
# → "Total: 9350.00 EUR"
```

El especificador `:,.2f` añade separador de miles (con coma, estilo anglosajón):

```python
big = f"Total: {48210.50:,.2f} EUR"
# → "Total: 48,210.50 EUR"
```

> [!nota]
> Las f-strings reemplazán a las funciones `CONCAT()`, `TEXT()` y al operador `&` de Excel. Son más legibles y menos propensas a errores que concatenar con `+`.

### Orden de evaluación (precedencia)

Python sigue el mismo orden que la aritmética estándar: paréntesis primero, luego `**`, luego `*`, `/`, `//`, `%`, y finalmente `+`, `-`. Igual que en Excel:

```python
# sin paréntesis: multiplicación primero
total = 4820.0 + 4820.0 * 0.21    # → 5832.2  (correcto si quieres base + IVA)

# con paréntesis: fuerza el orden
wrong = (4820.0 + 4820.0) * 0.21  # → 2024.4  (no es lo que querías)
```

Cuando dudes, añade paréntesis. Son gratis y eliminan ambigüedad.

## Ejemplo conducido

<!-- INLINE_QUIZ: division-type -->

```python
# datos de la factura de ACME, período 2026-04
invoice_id = "INV-001"
base_amount = 4820.0
tax_rate = 0.21
shipping_fee = 75.0

# cálculos
tax = base_amount * tax_rate
total = base_amount + tax + shipping_fee

# cuántos períodos completos de 30 días caben en 2026-04 (30 días)
days_in_period = 30
weeks_complete = days_in_period // 7    # cociente entero: 4
days_remainder = days_in_period % 7    # resto: 2

# construir el texto del resumen con f-string
summary = (
    f"Factura {invoice_id} | base {base_amount:.2f} EUR | "
    f"IVA {tax:.2f} EUR | total {total:.2f} EUR"
)
print(summary)
print(f"Semanas completas en el período: {weeks_complete} (resto: {days_remainder} días)")
```

Salida:

```
Factura INV-001 | base 4820.00 EUR | IVA 1012.20 EUR | total 5907.20 EUR
Semanas completas en el período: 4 (resto: 2 días)
```

## Tabla Excel ↔ Python

| Excel | Python | Notas |
|---|---|---|
| `=A1+B1` | `a + b` | igual |
| `=A1*0.21` | `a * 0.21` | igual |
| `=A1/B1` | `a / b` | siempre `float` en Python |
| `=INT(A1/B1)` | `a // b` | devuelve `int` |
| `=MOD(A1, B1)` | `a % b` | resto de la división |
| `=A1^2` | `a ** 2` | potencia |
| `=CONCAT(A1, " EUR")` | `f"{a} EUR"` o `str(a) + " EUR"` | f-string es preferible |
| `=TEXT(A1, "0.00")` | `f"{a:.2f}"` | especificador de formato |
| `=A1 & " " & B1` | `f"{a} {b}"` | f-string reemplaza `&` |

## Errores típicos

**1. Sumar string y número sin convertir.**
*Síntoma*: `TypeError: can only concatenate str (not "float") to str`.
*Causa*: `"INV-" + 42` falla porque `42` es `int`, no `str`.
*Arreglo*: `"INV-" + str(42)` o mejor `f"INV-{42}"`.

**2. Esperar que `/` devuelva entero.**
*Síntoma*: el resultado es `2.5` cuando esperabas `2`.
*Causa*: en Python 3, `/` siempre devuelve `float`. `5 / 2` es `2.5`.
*Arreglo*: si necesitas el cociente entero, usa `//`. Si necesitas el `float`, es correcto.

**3. Olvidar la `f` antes de la comilla.**
*Síntoma*: el string imprime las llaves literales: `"Factura {invoice_id} ..."`.
*Causa*: `"texto {variable}"` sin la `f` inicial es un string literal, no una f-string.
*Arreglo*: `f"texto {variable}"` con la `f` antes de la comilla.

**4. Usar `^` para potencia.**
*Síntoma*: `4 ^ 2` devuelve `6` en lugar de `16`.
*Causa*: en Python, `^` es el operador XOR bit a bit, no la potencia. `4 ^ 2` es `6` (XOR de los bits de 4 y 2).
*Arreglo*: `4 ** 2` para potencia.
