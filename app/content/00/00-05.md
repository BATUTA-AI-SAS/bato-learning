## Hilo conductor

Tienes VS Code abierto y un archivo `hola.py` creado. Antes de ejecutar archivos, hay un modo más inmediato: la **REPL** (Read-Eval-Print-Loop). Es el lugar donde Python evalúa cada línea en el momento y muestra el resultado. No necesitas instalar nada todavía: la versión de este sitio corre en el navegador con Pyodide, el mismo intérprete Python compilado para la web.

## Puente desde Excel

En Excel, cuando haces clic en una celda y escribes `=2+2`, Excel evalúa la expresión y muestra `4` en esa celda. La REPL hace exactamente lo mismo: escribes `2 + 2`, pulsas Enter, y Python responde `4`. La diferencia es que en Excel el resultado queda visible en la celda; en la REPL el resultado se imprime y desaparece de la pantalla a menos que lo asignes a un nombre (eso lo hace el siguiente módulo).

## Idea central

La **REPL** es el modo interactivo de Python. Al abrir la terminal y escribir `python3`, aparece el prompt `>>>`. Cada línea que escribes es una **expresión**: Python la evalúa, obtiene un **valor**, y lo imprime. El ciclo se repite hasta que escribes `exit()` o pulsas Ctrl-D. Sin crear archivos, sin instalar librerías, sin configurar nada: solo tú y el intérprete en conversación directa.

## Por qué importa

La REPL es tu primer banco de pruebas. Cuando no estés seguro de qué devuelve una operación o quieras explorar un tipo de dato, la REPL te da respuesta en menos de un segundo. Los desarrolladores Python con 10 años de experiencia siguen abriendo la REPL para verificar comportamientos puntuales antes de escribirlos en un archivo.

En los módulos siguientes usarás archivos `.py` para código que reutilizas. La REPL es para explorar, probar hipótesis, y entender cómo responde Python ante una expresión que no has visto antes. Distinguir cuándo usar la REPL y cuándo un archivo es un hábito que marca la diferencia entre programar eficientemente o perderse en archivos que aún no se necesitan.

## Cómo funciona por dentro

El nombre REPL describe exactamente el ciclo:

1. **Read** (leer): Python lee la línea que escribiste.
2. **Eval** (evaluar): Python calcula el valor de la expresión.
3. **Print** (imprimir): Python muestra el valor en pantalla.
4. **Loop** (repetir): vuelve a `>>>` y espera la siguiente línea.

Cuando escribes `2 + 2`, Python reconoce que `2`, `+`, y `2` forman una expresión aritmética, la evalúa internamente, obtiene el entero `4`, y lo imprime. Todo en milisegundos.

El prompt `>>>` es la señal de que Python está esperando input. Si escribes una expresión incompleta (por ejemplo, abres un paréntesis y no lo cierras), el prompt cambia a `...` para indicar que espera la continuación. Pulsa Enter en una línea vacía para que Python intente evaluar lo que tiene.

Los **tipos** que verás en esta primera sesión:

- `int` — número entero: `42`, `-3`, `0`. Sin punto decimal.
- `float` — número decimal: `4820.5`, `0.21`. Con punto. En prosa usarás coma (4.820,50 EUR); en código, punto (`4820.50`).
- `str` — texto (string): siempre entre comillas, simples o dobles. `"ACME"` y `'ACME'` son equivalentes. El texto puede contener cualquier carácter.
- `bool` — verdadero o falso: solo dos valores posibles, `True` o `False`. Con mayúscula inicial, siempre.

La función `type()` te dice el tipo de cualquier valor. `type(42)` devuelve `<class 'int'>`. Esa respuesta es el tipo del valor, no el valor en sí.

> [!nota]
> En la REPL del navegador (los ejercicios de este módulo), el resultado de cada expresión aparece automáticamente. En la REPL de terminal, igual. La diferencia con un archivo `.py`: en un archivo, el resultado de una expresión **no** se imprime solo; tienes que pedir explícitamente `print(expresion)`.

La función `len()` cuenta caracteres en un string: `len("factura")` devuelve `7`. También funciona con otros tipos que verás más adelante.

Para salir de la REPL de terminal: escribe `exit()` o pulsa Ctrl-D. En el navegador no hay que salir.

## Ejemplo conducido

<!-- INLINE_QUIZ: type-identification -->

Abre la REPL de terminal con `python3` o usa el editor de ejercicios abajo. Ejecuta estas expresiones una por una y observa los resultados:

```python
# aritmética básica
2 + 2

# operaciones con decimales (float)
4820.5 * 0.21

# texto (str): la empresa ACME tiene 4 caracteres
len("ACME")

# repetición de texto (ya la verás más en 00-07)
"ACME-" * 3

# tipo de un valor
type(4820.5)

# tipo de un string
type("factura INV-001")

# booleano: ¿4820 es mayor que 1000?
4820 > 1000
```

Salida esperada de cada línea (en orden):

```
4
1012.3049999999999
4
'ACME-ACME-ACME-'
<class 'float'>
<class 'str'>
True
```

> [!cuidado]
> El punto decimal en Python es siempre `.`, no `,`. Escribir `4.820,5` es un error de sintaxis. `4820.5` es el formato correcto en código.

## Tabla Excel ↔ Python

| En Excel | En la REPL de Python | Tipo resultante |
|---|---|---|
| Hacer clic en celda y escribir `=2+2` → Enter | `2 + 2` → Enter | `int` (`4`) |
| `=4820.5*0.21` en celda | `4820.5 * 0.21` en REPL | `float` |
| Celda con texto `ACME` | `"ACME"` | `str` |
| `=TRUE` en celda | `True` en REPL | `bool` |
| `=LEN("factura")` | `len("factura")` | `int` (`7`) |

## Errores típicos

**1. Texto sin comillas.**
*Síntoma*: `NameError: name 'ACME' is not defined`.
*Causa*: escribiste `ACME` sin comillas. Python lo interpreta como el nombre de una variable, no como texto.
*Arreglo*: `"ACME"` con comillas dobles o simples.

**2. Coma decimal en vez de punto.**
*Síntoma*: `SyntaxError` o un resultado inesperado.
*Causa*: escribiste `4.820,5` (formato de prosa español) en lugar de `4820.5`.
*Arreglo*: en código, el separador decimal es siempre `.`. Sin separador de miles.

**3. `True` con minúscula.**
*Síntoma*: `NameError: name 'true' is not defined`.
*Causa*: `true` y `false` no existen en Python. Los booleanos son `True` y `False` con mayúscula inicial.
*Arreglo*: `True` / `False`.
