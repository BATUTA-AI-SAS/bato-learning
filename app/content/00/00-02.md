## Hilo conductor

En `excel-to-python-bridge` (00-01) viste que programar es dar instrucciones a una máquina. Ahora viene la pregunta evidente: ¿dónde se escriben esas instrucciones para Python? No en Excel, no en un formulario web. Se escriben en la **terminal** — una ventana de texto donde el sistema operativo espera órdenes. Este módulo te lleva desde cero hasta ejecutar seis comandos básicos de navegación. Lo que se desbloquea en `files-folders-paths` (00-03): saber qué es una ruta y cómo decirle a la terminal exactamente dónde está tu archivo.

## Puente desde Excel

En Excel, la **barra de fórmulas** es el lugar donde escribes instrucciones que Excel ejecuta cuando presionas Enter. La terminal es exactamente eso, pero para el sistema operativo completo. Escribes una orden, presionas Enter, el sistema la ejecuta y muestra el resultado (si tiene alguno). La diferencia: en la barra de fórmulas las instrucciones son fórmulas que operan sobre celdas; en la terminal las instrucciones son **comandos** que operan sobre archivos, carpetas y procesos.

## Idea central

Una **terminal** es un programa que muestra texto y acepta texto. Adentro corre un **shell** — el programa que interpreta lo que escribes y lo ejecuta. El shell más común en macOS y Linux se llama **bash** o **zsh**; en Windows con WSL2 también es bash. El **prompt** es el texto que el shell muestra para indicar que está esperando una orden. Cuando el prompt aparece, el sistema está listo. Cuando no aparece, el sistema está ejecutando algo.

## Por qué importa

Python no tiene botón de «Ejecutar». El código en un archivo `.py` se ejecuta desde la terminal escribiendo `python archivo.py`. Toda la cadena de trabajo de este curso — escribir código, ejecutarlo, instalar paquetes, correr tests — pasa por la terminal. Sin terminal, no hay Python. Sin Python, no hay agentes de auditoría para ACME.

> [!cuidado]
> En Windows, la terminal del sistema (`cmd.exe` o PowerShell) usa comandos distintos a los de Linux y macOS. Este curso asume comandos Linux. **Si usas Windows, instala WSL2 con Ubuntu** antes de continuar. Las instrucciones están en la sección «Cómo funciona por dentro». En macOS y Linux no hay nada que instalar.

## Cómo funciona por dentro

### Abrir la terminal

**macOS**: busca «Terminal» en Spotlight (Cmd + Espacio) o en Aplicaciones → Utilidades. También puedes usar iTerm2 si lo tienes instalado.

**Linux (Ubuntu/Debian)**: Ctrl + Alt + T abre la terminal. O busca «Terminal» en el menú de aplicaciones.

**Windows**: instala WSL2 antes de seguir.
1. Abre PowerShell como administrador.
2. Ejecuta: `wsl --install`
3. Reinicia cuando te lo pida.
4. Abre «Ubuntu» desde el menú de inicio. La primera vez te pedirá crear un usuario y contraseña.
5. A partir de aquí, usa siempre la terminal de Ubuntu (WSL2), no la de Windows.

### Anatomía del prompt

Cuando abres la terminal, ves algo como esto:

```
usuario@maquina:~/proyectos$
```

Cuatro partes:

| Parte | Ejemplo | Qué significa |
|---|---|---|
| Usuario | `usuario` | Tu nombre de usuario en el sistema |
| Máquina | `maquina` | El nombre del equipo |
| Directorio actual | `~/proyectos` | La carpeta donde «estás» ahora mismo |
| Símbolo de prompt | `$` | «Listo para recibir una orden» |

El `$` es lo importante: cuando lo ves, escribes tu orden. Cuando no lo ves, el sistema está ocupado. Nunca copies el `$` cuando copies un comando de este curso — es parte del prompt, no del comando.

### Los seis comandos esenciales

**`pwd`** — *print working directory* — muestra en qué carpeta estás:
```
$ pwd
/home/usuario/proyectos
```

**`ls`** — *list* — lista los archivos y carpetas del directorio actual:
```
$ ls
acme-audit  facturas  notas.txt
```

Con el flag `-l` muestra más detalles (permisos, tamaño, fecha):
```
$ ls -l
drwxr-xr-x  3 usuario  staff   96 May 16 10:22 acme-audit
drwxr-xr-x  2 usuario  staff   64 May 15 09:00 facturas
-rw-r--r--  1 usuario  staff  412 May 14 08:30 notas.txt
```

**`cd`** — *change directory* — entra a una carpeta:
```
$ cd acme-audit
$ pwd
/home/usuario/proyectos/acme-audit
```

**`cd ..`** — sube una carpeta (el doble punto significa «carpeta padre»):
```
$ cd ..
$ pwd
/home/usuario/proyectos
```

**`mkdir nombre`** — *make directory* — crea una nueva carpeta:
```
$ mkdir reportes
$ ls
acme-audit  facturas  notas.txt  reportes
```

**`echo "texto"`** — imprime texto en la terminal:
```
$ echo "hola ACME"
hola ACME
```

### Argumentos y flags

Un **argumento** es información adicional que le pasas al comando. En `cd acme-audit`, el argumento es `acme-audit` — le dice a `cd` a qué carpeta entrar.

Un **flag** es una opción que modifica el comportamiento del comando. Empieza con `-`. En `ls -l`, el flag `-l` (de *long*) le dice a `ls` que muestre el formato largo con detalles.

Algunos comandos aceptan múltiples flags: `ls -la` muestra el formato largo (`-l`) y también los archivos ocultos (`-a`, los que empiezan con punto).

### Cómo se ve un error en la terminal

Si escribes un comando que no existe o un argumento incorrecto, la terminal muestra un error:

```
$ lss
command not found: lss

$ cd carpeta-inexistente
cd: no such file or directory: carpeta-inexistente
```

El error siempre describe el problema. Leerlo con atención es la mitad de la solución.

$ mkdir acme-audit
$ cd acme-audit
$ pwd

Cada línea que empieza con `$` es lo que escribes tú. Las líneas sin `$` son la respuesta del sistema.

## Tabla Excel ↔ Python

| Acción en Excel | Equivalente en terminal | Notas |
|---|---|---|
| Abrir un archivo `.xlsx` | `cd ruta/al/proyecto` + `python archivo.py` | En terminal primero navegas, luego ejecutas |
| Ver las hojas de un libro | `ls` (lista los archivos de la carpeta) | Los archivos son las «hojas» del proyecto |
| Crear un libro nuevo | `mkdir nombre-carpeta` | En Python los proyectos son carpetas |
| Barra de fórmulas | La línea de entrada del prompt | Escribes → Enter → resultado |
| Guardar con Ctrl+S | El archivo `.py` ya existe en disco al guardarlo en el editor | La terminal no guarda nada: ejecuta |

## Errores típicos del que viene de Excel

**1. Copiar el `$` del prompt junto con el comando.**
*Síntoma*: la terminal dice `command not found: $` o `$: command not found`.
*Causa*: en los tutoriales, el `$` indica que lo que sigue es un comando. No es parte del comando.
*Arreglo*: nunca copies el `$`. Solo copia lo que viene después.

**2. Pensar que `ls` muestra el contenido de los archivos.**
*Síntoma*: el lector ejecuta `ls notas.txt` esperando ver el contenido del archivo.
*Causa*: `ls` lista archivos y carpetas, no lee su contenido. Para leer el contenido de un archivo de texto se usa `cat archivo.txt` (o el editor de código, que verás en 00-04).
*Arreglo*: `ls` = ver qué hay en la carpeta. Para leer el archivo, usa el editor.

**3. Olvidar que la terminal tiene «posición».**
*Síntoma*: el lector ejecuta `python audit.py` y obtiene `No such file or directory`.
*Causa*: la terminal siempre está «parada» en alguna carpeta (el directorio actual). Si `audit.py` está en `acme-audit/` pero la terminal está en `proyectos/`, no lo encuentra.
*Arreglo*: ejecuta `pwd` para ver dónde estás, luego `cd` para moverte a la carpeta correcta. O usa la ruta completa (se explica en 00-03).

**4. Cerrar la terminal y perder el «lugar».**
*Síntoma*: el lector cierra y vuelve a abrir la terminal, y ya no está en el proyecto.
*Causa*: la terminal abre siempre en la carpeta personal (`~`). La «posición» no se guarda.
*Arreglo*: al abrir una terminal nueva, navega de nuevo a tu carpeta con `cd ~/ruta/al/proyecto`. Con práctica esto es automático.

## Salida esperada

Al terminar este módulo, puedes:

- Abrir una terminal en tu sistema operativo (macOS, Linux, o Windows con WSL2).
- Leer el prompt e identificar el directorio actual, el usuario y el símbolo `$`.

- Distinguir un argumento de un flag en un comando.
- Leer un mensaje de error de la terminal y extraer la causa del problema.
