## Hilo conductor

En `terminal-and-shell` (00-02) aprendiste a moverte por el sistema con `cd` y `ls`. Ahora viene el concepto que hace que esa navegación tenga sentido: las **rutas**. Una ruta es la dirección exacta de un archivo o carpeta en el disco. Sin entender rutas, no puedes decirle a Python dónde leer un CSV, dónde guardar un reporte o dónde buscar un módulo. Este módulo cubre eso. Lo que se desbloquea en `code-editor-vscode` (00-04): abrir la carpeta correcta del proyecto en VS Code.

## Puente desde Excel

Cuando guardas un archivo Excel, el cuadro de diálogo muestra algo como `C:\Users\maria\Documentos\facturas\acme_2026-04.xlsx` (Windows) o `/Users/maria/Documentos/facturas/acme_2026-04.xlsx` (macOS). Eso es una **ruta absoluta**: la dirección completa desde la raíz del disco hasta el archivo. Cuando le envías a un colega «está en la carpeta `facturas`, dentro de `Documentos`» — eso es una **ruta relativa**: parte desde algún punto conocido, no desde la raíz. Python usa exactamente los mismos conceptos.

## Idea central

Una **ruta** es la cadena de texto que identifica de forma única la ubicación de un archivo o carpeta en el sistema de archivos. Una **ruta absoluta** empieza desde la raíz del sistema (`/` en Linux/macOS, `C:\` en Windows) y especifica todos los niveles hasta el destino. Una **ruta relativa** parte desde el directorio actual (donde está parada la terminal) y usa `.` (aquí), `..` (carpeta padre) y nombres de carpeta para llegar al destino. La extensión de un archivo (`.py`, `.xlsx`, `.csv`) indica el formato del contenido — le ayuda al editor y al sistema operativo, no obliga al usuario.

## Por qué importa

Python necesita saber exactamente dónde leer y dónde escribir. Cuando escribes:

```python
with open("facturas/acme_2026-04.csv", "r") as f:
    ...
```

Python interpreta `"facturas/acme_2026-04.csv"` como ruta relativa al directorio desde donde se ejecutó el script. Si ejecutas el script desde la carpeta equivocada, Python no encuentra el archivo. Entender la diferencia entre ruta absoluta y relativa es la diferencia entre código que funciona siempre y código que funciona «en mi máquina».

## Cómo funciona por dentro

### El árbol de carpetas

El sistema de archivos es un árbol con una raíz. En Linux y macOS la raíz es `/`. En Windows es `C:\` (o la letra que corresponda). Todo archivo vive en una rama de ese árbol.

```
/
├── home/
│   └── maria/
│       ├── proyectos/
│       │   └── acme-audit/
│       │       ├── audit.py
│       │       └── facturas/
│       │           └── acme_2026-04.csv
│       └── notas.txt
└── usr/
    └── bin/
        └── python3
```

### Ruta absoluta

Empieza siempre con `/` (Linux/macOS) o la letra de unidad (Windows). Especifica todos los niveles:

```
/home/maria/proyectos/acme-audit/audit.py
/home/maria/proyectos/acme-audit/facturas/acme_2026-04.csv
```

Una ruta absoluta funciona desde cualquier directorio de trabajo — no importa dónde esté parada la terminal. Es como dar la dirección postal completa: calle, número, ciudad, país.

### Ruta relativa

Parte desde el directorio actual. Dos símbolos especiales:

- `.` — la carpeta actual (el directorio de trabajo).
- `..` — la carpeta padre (un nivel arriba).

Si la terminal está en `/home/maria/proyectos/acme-audit/`:

| Ruta relativa | Archivo que referencia |
|---|---|
| `audit.py` | `/home/maria/proyectos/acme-audit/audit.py` |
| `./audit.py` | Igual — el `.` es explícito pero redundante |
| `facturas/acme_2026-04.csv` | `/home/maria/proyectos/acme-audit/facturas/acme_2026-04.csv` |
| `../notas.txt` | `/home/maria/proyectos/notas.txt` (un nivel arriba) |
| `../../notas.txt` | `/home/maria/notas.txt` (dos niveles arriba) |

### El símbolo `~` (home)

`~` es una abreviación del directorio personal del usuario. En Linux suele ser `/home/maria`; en macOS, `/Users/maria`. Siempre funciona:

```bash
$ cd ~
$ pwd
/home/maria

$ cd ~/proyectos/acme-audit
$ pwd
/home/maria/proyectos/acme-audit
```

`~` solo funciona en la shell — no en código Python. En Python se usa `pathlib.Path.home()` para obtener la carpeta personal (lo verás en módulos posteriores).

### La extensión de un archivo

La extensión es la parte del nombre después del último punto: `.py`, `.xlsx`, `.csv`, `.md`, `.yaml`, `.json`.

| Extensión | Contenido típico | Ejecuta... |
|---|---|---|
| `.py` | Código Python | `python archivo.py` |
| `.xlsx` | Libro Excel | Excel, LibreOffice |
| `.csv` | Datos separados por coma | Cualquier editor de texto, Python, Excel |
| `.md` | Markdown (texto formateado) | Cualquier editor de texto, GitHub |
| `.yaml` | Configuración legible | Editor de texto, aplicaciones que lo consumen |
| `.json` | Datos estructurados | Editor de texto, Python, APIs |

> [!nota]
> La extensión no obliga al sistema — es una convención. Puedes nombrar un archivo Python `audit.txt` y Python lo ejecutará igual si le dices explícitamente que lo haga. Pero no lo hagas: la extensión le dice a tu editor qué colores usar, y a ti qué esperar al abrirlo.

### Convenciones para nombres de archivo en Python

- **snake_case**: `audit_run.py`, `acme_2026_04.csv`. Sin espacios, sin guiones (salvo en proyectos con convención propia).
- **Sin espacios**: un espacio en el nombre de un archivo complica los comandos de terminal. `mi archivo.py` requiere comillas: `python "mi archivo.py"` — evitable.
- **Sin acentos ni caracteres especiales** en nombres de archivos Python: `análisis.py` puede dar problemas en algunos sistemas. Usa `analisis.py`.

## Ejemplo conducido

Estructura de carpetas del proyecto de auditoría de ACME:

```
~/proyectos/acme-audit/
├── audit.py
├── facturas/
│   ├── acme_2026-04.csv
│   └── acme_2026-03.csv
└── reportes/
    └── acme_2026-04_report.md
```

Con la terminal en `~/proyectos/acme-audit/`:

```bash
$ pwd
/home/maria/proyectos/acme-audit

$ ls
audit.py  facturas  reportes

$ ls facturas
acme_2026-04.csv  acme_2026-03.csv

# referencia relativa desde el directorio actual:
$ python audit.py
# audit.py puede abrir "facturas/acme_2026-04.csv" sin ruta absoluta

# subir un nivel y volver:
$ cd ..
$ pwd
/home/maria/proyectos

$ ls acme-audit/facturas
acme_2026-04.csv  acme_2026-03.csv
```

En `audit.py`, la ruta relativa al CSV:
```python
# si el script se ejecuta desde ~/proyectos/acme-audit/
with open("facturas/acme_2026-04.csv", "r") as f:
    ...
```

## Tabla Excel ↔ Python

| Concepto Excel | Equivalente Python | Notas |
|---|---|---|
| Ruta del archivo en «Guardar como» | Ruta absoluta (`/home/maria/...`) | Misma idea, distinta interfaz |
| «Está en la carpeta de facturas» | Ruta relativa (`facturas/archivo.csv`) | Depende del directorio de trabajo |
| Extensión `.xlsx` | Extensión `.py` | Indica el formato; convención, no obligación técnica |
| Nombre con espacios `mi reporte.xlsx` | Evitar espacios en archivos `.py` | En terminal, los espacios complican los comandos |
| Carpeta «Mis documentos» | `~` (home) en terminal, `Path.home()` en Python | La carpeta personal del usuario |

## Errores típicos del que viene de Excel

**1. Usar la ruta absoluta hardcodeada de su propia máquina.**
*Síntoma*: el script funciona en la máquina del desarrollador pero falla en la del colega con `FileNotFoundError: /home/maria/proyectos/acme-audit/...`.
*Causa*: la ruta absoluta `/home/maria/...` solo existe en la máquina de Maria.
*Arreglo*: usar rutas relativas al directorio del proyecto, o construir la ruta con `pathlib` usando `Path(__file__).parent` (se ve en módulos posteriores).

**2. Confundir `.` y `..`.**
*Síntoma*: el lector escribe `../audit.py` pensando que sube al archivo que está en el mismo nivel.
*Causa*: `..` sube un nivel en el árbol de carpetas, no «va al lado». `./audit.py` y `audit.py` son equivalentes.
*Arreglo*: dibujar el árbol de carpetas en papel y trazar la ruta desde el directorio actual hasta el destino.

**3. Usar espacios en nombres de archivos Python.**
*Síntoma*: `python mi script.py` falla con `can't open file 'mi'`.
*Causa*: la terminal interpreta el espacio como separador de argumentos. Lee `mi` como el nombre del archivo y `script.py` como otro argumento.
*Arreglo*: usar `python "mi script.py"` (con comillas) o, mejor, renombrar el archivo a `mi_script.py`.

**4. Pensar que la extensión cambia lo que el archivo contiene.**
*Síntoma*: el lector renombra `audit.txt` a `audit.py` esperando que «se convierta» en Python.
*Causa*: la extensión es solo parte del nombre. El contenido del archivo no cambia al renombrarlo.
*Arreglo*: la extensión indica el formato; el contenido real determina si Python puede ejecutarlo. Un `.py` con texto de Word dentro no funciona.
