## Hilo conductor

En 00-11 aprendiste a guardar muchas facturas en una lista de dicts. Ahora necesitas procesarlas todas: sumar montos, filtrar por estado, identificar proveedores. En Excel arrastras una fórmula hacia abajo para aplicarla a cada fila. En Python, el bucle `for` hace lo mismo — pero el «cuerpo» puede contener cualquier lógica, no solo una fórmula. Después de este módulo, 00-13 te enseña a empaquetar esa lógica en funciones reutilizables.

## Puente desde Excel

| Excel | Python | Diferencia |
|-------|--------|------------|
| Arrastrar `=A2*0.21` desde A2 hasta A1000 | `for inv in invoices:` + operación | En Python la lógica puede ser más rica: condicionales, múltiples líneas, llamadas a funciones |
| `=SUMA(A2:A1000)` | Acumulador `total += inv["amount"]` | El bucle te da control fila a fila |
| `=SUMIF(B2:B1000,"acme",A2:A1000)` | `for` + `if` dentro | Más legible cuando la condición es compleja |
| Numerar filas con A1=1, A2=2, ... | `enumerate(invoices)` | El índice viene junto con el elemento |

La ventaja del `for` sobre arrastrar fórmulas no es la velocidad (en hojas pequeñas Excel es perfectamente válido). La ventaja es que la lógica puede bifurcarse, llamar a otras funciones, construir estructuras nuevas — todo en las mismas líneas.

## Idea central

`for elemento in colección:` repite el bloque indentado bajo él, una vez por cada elemento de la colección. Cada repetición, la variable `elemento` toma el valor del siguiente ítem. Cuando se acaban los ítems, el bucle termina. El bloque puede ser tan simple como un `print` o tan complejo como docenas de líneas con condicionales y llamadas a funciones.

## Por qué importa

Sin `for`, cualquier operación sobre un dataset requiere duplicar el código manualmente o depender de funciones de hoja de cálculo con sus limitaciones. Con `for`, procesas cien facturas exactamente igual que tres: el mismo código, sin cambios.

En la auditoría de ACME del período `"2026-04"`, el script recorre todas las facturas, suma los montos válidos, acumula las problemáticas y genera un reporte. Sin `for`, eso no es automatizable.

## Cómo funciona por dentro

### El bucle básico

```python
vendors = ["acme_global", "andina_co", "globex_log"]

for vendor in vendors:
    print(vendor)
# imprime:
# acme_global
# andina_co
# globex_log
```

La variable `vendor` existe dentro y fuera del bucle. Al terminar el bucle, contiene el **último** valor que tomó (`"globex_log"`).

### Iterar una lista de diccionarios

El patrón más frecuente en datos: cada elemento del `for` es un dict.

```python
invoices = [
    {"id": "INV-001", "vendor": "acme_global",  "amount": 4820.0},
    {"id": "INV-002", "vendor": "andina_co",     "amount": 1500.0},
    {"id": "INV-003", "vendor": "globex_log",    "amount": 3100.0},
]

for inv in invoices:
    print(f"{inv['id']}: {inv['amount']} EUR")
```

### Acumulador

Para sumar, contar o construir algo a partir de los elementos, declaras una variable antes del bucle y la actualizas dentro:

```python
total = 0.0

for inv in invoices:
    total = total + inv["amount"]
    # o equivalentemente: total += inv["amount"]

print(f"total: {total} EUR")   # 9420.0
```

Este es el patrón exacto que `=SUMA()` de Excel hace internamente: recorre las celdas del rango y acumula.

### Filtrar con `if` dentro del bucle

Combinas `for` con `if` para procesar solo los elementos que cumplen una condición. Esto equivale a `=SUMIF()` pero más flexible:

```python
total_unpaid = 0.0
unpaid_invoices = []

for inv in invoices:
    if not inv.get("paid", False):
        total_unpaid += inv["amount"]
        unpaid_invoices.append(inv["id"])

print(f"pendiente: {total_unpaid} EUR")
print(f"facturas: {unpaid_invoices}")
```

> [!cuidado]
> No modifiques la lista que estás iterando dentro del `for` — no uses `.remove()` o `.append()` sobre la misma lista que recorres. Si necesitas construir una lista filtrada, crea una lista nueva (como `unpaid_invoices = []` arriba) y añade a ella.

### `range(N)`: iterar N veces

Cuando no tienes una lista de elementos sino que simplemente quieres repetir algo N veces:

```python
for i in range(5):
    print(i)
# 0, 1, 2, 3, 4
```

`range(N)` genera los números de 0 a N-1. `range(2, 5)` genera 2, 3, 4. Úsalo cuando el número de repeticiones importa, no cuando tienes una lista de datos.

### `enumerate`: índice y valor juntos

Cuando necesitas saber la posición del elemento además de su valor:

```python
for i, inv in enumerate(invoices):
    print(f"#{i+1} — {inv['id']}: {inv['amount']} EUR")
# #1 — INV-001: 4820.0 EUR
# #2 — INV-002: 1500.0 EUR
# #3 — INV-003: 3100.0 EUR
```

`enumerate` devuelve pares `(índice, elemento)` que desempaquetas en dos variables. El índice empieza en 0; `i+1` lo muestra como número de fila legible. 

> [!nota]
> El anti-patrón equivalente sería `for i in range(len(invoices)): inv = invoices[i]`. Funciona, pero `enumerate` es más directo, más legible y elimina la posibilidad de errores de índice. En Track A verás por qué.

### Iterar un diccionario

`for key in my_dict:` itera sobre las **claves**. Para claves y valores a la vez, usa `.items()`:

```python
invoice = {"id": "INV-001", "vendor": "acme_global", "amount": 4820.0}

for key in invoice:
    print(key)           # "id", "vendor", "amount"

for key, value in invoice.items():
    print(f"{key}: {value}")
```

Pipeline completo: recorre las facturas de ACME del período `"2026-04"`, calcula el total de facturas válidas, identifica las no pagadas y detecta las de monto cero.

```python
invoices = [
    {"id": "INV-001", "vendor": "acme_global",  "amount": 4820.0, "paid": True},
    {"id": "INV-002", "vendor": "andina_co",     "amount": 1500.0, "paid": False},
    {"id": "INV-003", "vendor": "globex_log",    "amount": 3100.0, "paid": False},
    {"id": "INV-004", "vendor": "acme_global",   "amount": 0.0,    "paid": False},
]

total_valid = 0.0
unpaid = []
zero_amount = []

for inv in invoices:
    if inv["amount"] == 0.0:
        zero_amount.append(inv["id"])
        continue        # salta el resto del cuerpo para esta iteración

    total_valid += inv["amount"]

    if not inv["paid"]:
        unpaid.append(inv["id"])

print(f"total válido: {total_valid} EUR")   # 9420.0 EUR
print(f"no pagadas: {unpaid}")              # ['INV-002', 'INV-003']
print(f"monto cero: {zero_amount}")         # ['INV-004']
```

`continue` salta al siguiente elemento del bucle sin ejecutar el resto del cuerpo. Útil para descartar casos especiales al inicio y mantener el camino principal limpio.

## Determinístico vs agéntico

| Tramo | Tipo | Por qué |
|-------|------|---------|
| Sumar montos con acumulador | determinístico | Operación aritmética cerrada; mismo input → mismo output |
| Filtrar facturas por `paid == False` | determinístico | Condición booleana exacta |
| Detectar monto cero | determinístico | Comparación numérica; regla cerrada |
| Determinar si una factura no pagada es disputa o simple retraso | agéntico | Requiere contexto del cliente y del proveedor; no hay regla que lo cubra siempre |

## Errores típicos

**1. Modificar la lista mientras se itera.**
*Síntoma*: el bucle se salta elementos o lanza `RuntimeError: list changed size during iteration`.
*Causa*: llamar a `.remove()` o `.append()` sobre la lista que el `for` está recorriendo.
*Arreglo*: construye una lista nueva dentro del bucle y trabaja con ella después.

**2. Acumulador no inicializado.**
*Síntoma*: `NameError: name 'total' is not defined` o acumulador con valor incorrecto.
*Causa*: olvidar declarar `total = 0.0` antes del `for`.
*Arreglo*: siempre inicializa la variable acumuladora antes del bucle.

**3. Usar `range(len(lista))` cuando se necesita el elemento.**
*Síntoma*: `for i in range(len(invoices)): inv = invoices[i]` — funciona pero es innecesariamente complejo.
*Causa*: hábito de otros lenguajes o de Excel (trabajar con índices numéricos).
*Arreglo*: `for inv in invoices:` si no necesitas el índice; `for i, inv in enumerate(invoices):` si lo necesitas.

**4. Variable del bucle usada fuera con valor inesperado.**
*Síntoma*: después del `for`, `inv` tiene el valor de la última factura, no la que esperabas.
*Causa*: la variable del bucle conserva su último valor al terminar.
*Arreglo*: si necesitas el resultado del bucle, guárdalo en una variable separada dentro del cuerpo.
