## Hilo conductor

En 00-05 evaluaste expresiones en la REPL y los resultados desaparecieron. Para reutilizar un valor, necesitas guardarlo bajo un nombre. Eso es una **variable**. Este módulo cubre qué es exactamente una variable en Python, qué tipos de valores puede guardar, y cómo Python trata la reasignación.

## Puente desde Excel

En Excel puedes darle nombre a una celda: seleccionas `B4`, vas al cuadro de nombres (arriba a la izquierda), escribes `monto`, y desde ese momento `=monto * 0.21` funciona en cualquier fórmula. En Python, `monto = 4820.0` hace algo similar: el nombre `monto` queda ligado al valor `4820.0` y puedes usarlo en las líneas siguientes. La diferencia clave: en Excel el nombre apunta a una coordenada física (`B4`). En Python el nombre apunta directamente al valor, no a una posición de memoria fija.

## Idea central

Una **variable** en Python es una etiqueta que apunta a un valor. La **asignación** `monto = 4820.0` crea esa etiqueta y la hace apuntar al float `4820.0`. Si después escribes `monto = 9000.0`, la etiqueta ahora apunta a `9000.0`. El valor `4820.0` anterior no cambia; simplemente ya no tiene ninguna etiqueta que lo señale. Python lo borrará de memoria en algún momento. Los valores tienen **tipos** (`int`, `float`, `str`, `bool`, `None`) que determinan qué operaciones pueden hacerse con ellos.

## Por qué importa

Sin variables, cada cálculo empieza de cero. Con variables, puedes expresar la lógica de negocio de forma legible:

```python
amount = 4820.0
tax_rate = 0.21
total = amount * (1 + tax_rate)
```

Tres líneas que cualquier persona del equipo de finanzas puede auditar. Si el tipo impositivo cambia al 23 %, cambias una sola línea: `tax_rate = 0.23`. En Excel, habrías buscado todas las celdas con `0.21` y cambiado cada una por separado.

El tipo de una variable también importa para detectar errores antes de que lleguen a producción. Si `amount` es un string `"4820"` en lugar de un float `4820.0`, la multiplicación `amount * tax_rate` fallará con `TypeError`. Conocer los tipos es el primer paso para entender por qué un script falla.

## Cómo funciona por dentro

### La asignación no es una ecuación matemática

`monto = 4820.0` se lee «monto recibe 4820.0». No es una afirmación de que monto es igual a 4820.0 para siempre. Es una instrucción: «crea la etiqueta `monto` y apúntala al valor `4820.0`».

El operador `=` en Python es unidireccional: el valor de la derecha se evalúa primero, y luego se asigna a la etiqueta de la izquierda. Por eso `monto = monto + 100.0` es válido: Python evalúa `monto + 100.0` (obteniendo `4920.0`), y luego re-apunta la etiqueta `monto` a ese nuevo valor.

### Los cinco tipos básicos

**`int`** — número entero, sin punto decimal. `42`, `-3`, `0`, `1_000_000` (el guión bajo es separador visual, Python lo ignora). Operaciones: `+`, `-`, `*`, `//`, `%`, `**`. La división `/` entre dos `int` produce un `float`.

**`float`** — número con punto decimal. `4820.5`, `0.21`, `-1.5`. Para finanzas, en Track C usarás `decimal.Decimal` para montos exactos. Por ahora, `float` es suficiente.

<details>
<summary>Para profundizar: precisión de números decimales (opcional)</summary>

Representación interna: punto flotante binario IEEE 754. Esto implica que `0.1 + 0.2` no es exactamente `0.3` (es `0.30000000000000004`). Para cálculos monetarios con centavos exactos, usa `decimal.Decimal` en lugar de `float` — lo verás en Track C.

</details>

**`str`** — texto. Cualquier secuencia de caracteres entre comillas simples o dobles. `"INV-001"`, `'ACME'`, `"período 2026-04"`. Las comillas no forman parte del valor: `"ACME"` contiene cuatro caracteres, no seis. Los strings son **inmutables**: no puedes cambiar un carácter individual, solo crear un string nuevo.

**`bool`** — booleano. Solo dos valores: `True` y `False`, con mayúscula. Úsalos para condiciones lógicas.

<details>
<summary>Para profundizar: conversión implícita y truthy/falsy (opcional)</summary>

`True` es equivalente a `1` y `False` a `0` en contextos numéricos. Más allá de eso, Python trata muchos valores como "falsy": `0`, `""`, `[]`, `{}`, `None` son todos `False` en un `if`. Cualquier número distinto de cero, string no vacío o lista con elementos son "truthy". Es útil saberlo para leer código ajeno, pero en este curso usa comparaciones explícitas (`valor == 0`, `valor is None`) para que el código sea claro.

</details>

**`None`** — el valor vacío intencional. No es `0`, no es `""`, no es `False`. Es la señal explícita de «este valor no existe o no se sabe». Equivale a una celda realmente vacía en Excel, a diferencia de una celda con `0`.

### Nombres de variables: convenciones

El curso usa **inglés** para todos los identificadores. La convención es `snake_case`: palabras en minúsculas separadas por guión bajo.

- Correcto: `unit_price`, `invoice_id`, `tax_rate`, `is_paid`
- Incorrecto: `PrecioUnitario`, `factura_id`, `TaxRate`, `isPaid`

Los nombres no pueden empezar por número, no pueden tener espacios, y no pueden ser palabras reservadas de Python (`if`, `for`, `True`, `None`, etc.).

### Inspeccionar el tipo

`type(valor)` devuelve el tipo del valor. En la REPL:

```python
>>> type(4820.0)
<class 'float'>
>>> type("INV-001")
<class 'str'>
>>> type(True)
<class 'bool'>
>>> type(None)
<class 'NoneType'>
```

`isinstance(valor, tipo)` devuelve `True` si el valor es de ese tipo (o una subclase):

```python
>>> isinstance(4820.0, float)
True
>>> isinstance("INV-001", str)
True
```

<details>
<summary>Para profundizar: inmutabilidad, id() e identidad de objetos (opcional)</summary>

En Python, los enteros y los flotantes son objetos inmutables. Cuando reasignas `monto = 9000.0`, no estás modificando el objeto `4820.0`. Estás moviendo la etiqueta `monto` para que apunte a un objeto nuevo. Puedes verificar esto con `id(monto)` antes y después de la reasignación: obtendrás números distintos. El operador `is` compara identidad (mismo objeto en memoria), mientras que `==` compara valor. Usa `is` solo para comparar con `None` (`valor is None`). Este detalle importa más con tipos mutables (`list`, `dict`) que verás en 00-11.

</details>

```python
# importe final

```

## Errores típicos

**1. Usar la variable antes de asignarla.**
*Síntoma*: `NameError: name 'total' is not defined`.
*Causa*: intentas usar `total` en una expresión antes de ejecutar `total = ...`.
*Arreglo*: asegúrate de que la línea de asignación aparece antes de la línea que usa la variable.

**2. Confundir `=` con `==`.**
*Síntoma*: el código no hace lo que esperas, o `SyntaxError` en la izquierda.
*Causa*: `=` asigna un valor a una etiqueta. `==` compara dos valores y devuelve `True` o `False`. Son operaciones distintas.
*Arreglo*: en condiciones (`if`), usa `==`. Para guardar un valor, usa `=`.

**3. Nombre de variable con acento o ñ.**
*Síntoma*: el código funciona en tu máquina pero falla en servidores con encoding diferente.
*Causa*: Python 3 acepta Unicode en identificadores, pero los acentos y la ñ generan problemas en equipos con configuraciones distintas.
*Arreglo*: nombres en inglés, `snake_case`, sin acentos ni caracteres especiales.

**4. `None` vs `0` vs `""`.**
*Síntoma*: una condición que comprueba `notas == ""` no detecta `None`.
*Causa*: `None`, `0`, y `""` son valores distintos. Ninguno es igual al otro.
*Arreglo*: para comprobar `None`, usa `valor is None`. Para `""`, usa `valor == ""`.
