## Hilo conductor

En `files-folders-paths` (00-03) aprendiste a navegar el sistema de archivos. Ahora necesitas una herramienta para crear y editar los archivos `.py`. Excel es editor y ejecutor a la vez — abre la hoja y puedes escribir fórmulas y verlas calcular en el mismo lugar. Python separa los dos roles: hay un **editor** para escribir, y hay la **terminal** para ejecutar. Este módulo cubre el editor. Lo que se desbloquea en `first-python-repl` (00-05): ejecutar tu primera instrucción Python real, con Python instalado y el editor listo.

## Puente desde Excel

Excel combina en una sola interfaz el editor (donde escribes fórmulas), el motor de cálculo (que evalúa las fórmulas) y la visualización (la cuadrícula). Python **separa** estos tres roles:

- El **editor** (VS Code) es donde escribes el código. No lo ejecuta.
- El **intérprete** (Python) es el motor que evalúa el código.
- La **terminal** es donde ves el resultado.

Esta separación parece un paso atrás al principio. En realidad es lo que permite escalar: el mismo código puede ejecutarse en tu laptop, en un servidor en la nube y en un proceso automático sin tocar el editor.

## Idea central

Un **editor de código** es un programa para escribir y editar archivos de texto con asistencia para código: colorea palabras clave (**syntax highlighting**), sugiere completado automático y muestra errores antes de ejecutar. **VS Code** (Visual Studio Code, de Microsoft) es el editor más común para Python en 2026: gratuito, extensible, disponible en macOS, Linux y Windows. Un **IDE** (*Integrated Development Environment*) es un editor con más herramientas integradas — debugger, terminal, gestión de proyecto — como PyCharm. Para este curso, VS Code es suficiente.

## Por qué importa

Sin editor, escribir Python en el bloc de notas del sistema es posible pero lento y propenso a errores silenciosos. Un error de sangría (indentación) en Python no es un error de estilo — es un error de sintaxis que hace fallar el código. El editor muestra esa sangría en color, y VS Code la corrige automáticamente al presionar Enter después de un bloque. Esos segundos ahorrados en cada línea se acumulan en horas a lo largo del curso.

## Cómo funciona por dentro

### Instalar VS Code

1. Ve a [https://code.visualstudio.com](https://code.visualstudio.com).
2. Descarga el instalador para tu sistema operativo.
3. Instala con las opciones por defecto.

En Linux (Ubuntu/Debian) también puedes instalar desde la terminal:
```bash
sudo snap install code --classic
```

> [!nota]
> Si prefieres **Cursor** (un fork de VS Code con IA integrada) o **Sublime Text**, ambos sirven para este curso. Lo que se describe aquí aplica en su mayoría a cualquier editor moderno. Pero VS Code es el que verás en los ejemplos.

### Abrir una carpeta de proyecto

VS Code trabaja con **workspaces** — una carpeta que contiene todo el proyecto. Para abrir el proyecto de ACME:

1. Abre VS Code.
2. Menú «File» → «Open Folder» (macOS: «File» → «Open...»).
3. Navega a `~/proyectos/acme-audit` y selecciónala.

O desde la terminal, con VS Code instalado:
```bash
$ cd ~/proyectos/acme-audit
$ code .
```

El punto `.` le dice a VS Code que abra la carpeta actual. El panel izquierdo («Explorer») muestra el árbol de archivos del proyecto.

### Crear un archivo nuevo

En el Explorer (panel izquierdo):
1. Haz clic derecho sobre la carpeta → «New File».
2. Escribe `hola.py` y presiona Enter.

O con el atajo: Ctrl+N (Cmd+N en macOS) crea un archivo temporal. Para guardarlo con nombre: Ctrl+S (Cmd+S).

### Syntax highlighting

VS Code detecta el tipo de archivo por la extensión. Al crear `hola.py`, colorea automáticamente:

- **azul/naranja**: palabras clave del lenguaje (`print`, `def`, `if`, `for`).
- **verde**: strings — texto entre comillas (`"hola ACME"`).
- **gris**: comentarios — texto después de `#`.
- **blanco/claro**: nombres de variables y funciones que tú defines.

Este coloreado no cambia lo que Python ejecuta. Es solo información visual para el programador.

### Escribir el primer archivo

En `hola.py`, escribe esto:
```python
# primer archivo Python de ACME
nombre_empresa = "ACME"
print(f"hola {nombre_empresa}")
```

Al guardar (Ctrl+S), VS Code colorea cada parte. Observa:
- `# primer archivo...` → en gris (es un comentario, Python lo ignora).
- `nombre_empresa` → en blanco (nombre de variable).
- `"ACME"` → en verde (string).
- `print` → en azul (función nativa).
- `f"hola {nombre_empresa}"` → verde con `{nombre_empresa}` diferenciado (f-string con interpolación).

Para ejecutar este archivo, vas a la terminal y escribes:
```bash
$ python3 hola.py
hola ACME
```

El editor no ejecuta nada. La terminal sí.

### Terminal integrada en VS Code

VS Code tiene una terminal integrada (Ctrl+\` o menú «Terminal» → «New Terminal»). Abre la terminal dentro del editor, ya posicionada en la carpeta del workspace. Es la misma terminal de 00-02 — solo más conveniente porque está dentro del editor.

### Lo que NO se configura todavía

- Extensiones de Python (pylance, python extension pack): llegan en 00-08 cuando instales Python.
- Linting y formateo automático: llega en A01.
- Debugger integrado: llega cuando el código sea suficientemente complejo para justificarlo.

> [!cuidado]
> VS Code puede pedirte instalar la «Python Extension» al abrir un `.py`. Puedes instalarla ahora — no hace daño. Pero si Python no está instalado aún (llega en 00-08), el editor mostrará advertencias sobre el intérprete. Son normales en este punto.

## Ejemplo conducido

Flujo completo desde cero hasta ver el primer output:

```bash
# en la terminal, crear la carpeta y el archivo
$ mkdir -p ~/proyectos/acme-audit
$ cd ~/proyectos/acme-audit
$ code .          # abre VS Code en esta carpeta
```

En VS Code:
1. Crear `hola.py` (clic derecho en Explorer → New File).
2. Escribir:
```python
# auditoría ACME, período 2026-04
periodo = "2026-04"
empresa = "ACME"
print(f"iniciando auditoría de {empresa} para el período {periodo}")
```
3. Guardar: Ctrl+S.

En la terminal integrada de VS Code (Ctrl+\`):
```bash
$ python3 hola.py
iniciando auditoría de ACME para el período 2026-04
```

El archivo existe en disco. El editor lo abrió para editar. La terminal lo ejecutó. Tres roles separados, cada uno haciendo lo suyo.

## Tabla Excel ↔ Python

| Concepto Excel | Equivalente Python | Diferencia |
|---|---|---|
| Excel (editor + ejecutor + visualizador) | VS Code (editor) + Python (ejecutor) + terminal (visualizador) | Python separa los tres roles |
| Editar una celda | Editar un archivo `.py` en VS Code | En Python, el «contenedor» es el archivo, no la celda |
| Colorear celdas para destacar errores | Syntax highlighting de VS Code | El editor ayuda a leer; no garantiza que el código esté bien |
| Guardar el libro (Ctrl+S) | Guardar el archivo (Ctrl+S) | Igual. Sin guardar, los cambios no están en disco |
| Abrir un libro reciente | `code .` desde la terminal en la carpeta del proyecto | VS Code recuerda workspaces recientes en «File → Open Recent» |
| VBA Editor (para macros) | VS Code con archivo `.py` | VS Code es para todo Python; el VBA Editor es solo para macros |

## Errores típicos del que viene de Excel

**1. Esperar que al guardar el archivo se ejecute el código.**
*Síntoma*: el lector guarda `hola.py` y espera ver el resultado en el editor.
*Causa*: Excel recalcula al guardar. VS Code no ejecuta al guardar — solo escribe el archivo en disco.
*Arreglo*: guardar con Ctrl+S y luego ejecutar en la terminal con `python3 hola.py`. Dos pasos separados.

**2. Confundir el editor con la terminal integrada.**
*Síntoma*: el lector escribe `python3 hola.py` en el panel de edición del archivo, no en la terminal.
*Causa*: VS Code tiene dos áreas de texto: el editor (para código) y la terminal (para comandos). Se confunden al principio.
*Arreglo*: la terminal está en la parte inferior de VS Code (Ctrl+\`). El cursor parpadeante con el prompt `$` indica que es la terminal.

**3. Abrir el archivo `.py` directamente sin abrir la carpeta.**
*Síntoma*: el Explorer de VS Code muestra solo el archivo suelto, sin árbol de carpetas.
*Causa*: al hacer doble clic en el archivo, VS Code lo abre sin cargar el workspace.
*Arreglo*: usar «File → Open Folder» (no «Open File») para abrir la carpeta del proyecto. El árbol de carpetas es esencial para proyectos con múltiples archivos.

**4. Creer que el coloreado indica que el código está correcto.**
*Síntoma*: el lector ve el código coloreado y asume que no hay errores.
*Causa*: el syntax highlighting colorea la estructura del lenguaje, no la lógica.
*Arreglo*: el coloreado muestra la estructura sintáctica. Los errores lógicos (código que corre pero hace algo incorrecto) solo aparecen al ejecutar el código.

- Distinguir el panel de edición de la terminal integrada en VS Code.
